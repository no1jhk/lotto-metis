---

## 앱 정체성

> "무작위가 아님을 데이터로 증명하는 로또 분석 앱"

로또는 예측 불가능하지만, 다른 구매자보다 유리한 조합을 선택하는 것은 가능하다.
LottoMetis는 그 과정을 데이터로 자동화하고, 결과를 장기 추적으로 검증한다.

---

## 핵심 가치

**1. 설명 가능한 번호 생성**
단순 빈도 통계가 아니라 "왜 이 번호인가"를 설명하는 근거 카드 제공

**2. 구매-추적-검증 루프**
번호 생성 → 구매 등록 → 자동 당첨 확인 → 성과 누적 → 전략 개선

**3. 장기 데이터 축적**
6개월, 1년 이상 쌓인 성과 데이터가 앱의 신뢰성을 만든다

## 아이디어 보관함

> 작업 중 생기는 아이디어 — 검토 전 메모

### 기능 아이디어
- 군중 회피 전략: 사람들이 많이 사는 번호 = 당첨금 낮아짐 → 역으로 피하기
  (LottoPick 분석에서 착안 / 기대값 점수에 통합 가능)
- 미출현 기간 차트: 오래 안 나온 번호 시각화 (구현 쉽고 임팩트 있음)
- 당첨금 기대값 점수: 번호 조합의 "희귀도" → 당첨 시 받을 금액 예측
- 이번 주 AI 브리핑 카드: 매주 토요일 Claude API로 패턴 요약 자동 생성
- 보너스 번호 후보 별도 표시 (2등 전략 특화)

### 비주얼 아이디어
- 히어로 비주얼: 골든 돼지저금통 3D (현재 Three.js 구현 시도 중)
  → GLB 파일 확보 시 즉시 교체 가능 구조로 준비
- 번호 생성 시 볼이 굴러나오는 애니메이션 (Spline or Three.js)
- 히트맵: 시간축 있는 인터랙티브 히트맵 (가로: 시간, 세로: 번호 1~45)

### UX 아이디어 — 동행복권 용지 UI 참고 (2026-03-17)
  ref: https://el.dhlottery.co.kr/game/TotalGame.jsp?LottoId=LO40
- 번호 입력을 인풋 6개 → 1~45 격자 클릭 방식으로 교체
  → 동행복권 용지 UI와 동일한 UX → 사용자에게 친숙하고 직관적
  → 번호 클릭 시 해당 칸 하이라이트 (골드 색상)
- 생성된 번호를 A~E 게임 슬롯으로 표시 (현재 #1~#5 → A~E)
- 번호 생성 버튼 클릭 시 격자에서 번호가 순서대로 선택되는 애니메이션
  → 실제 로또 용지 마킹하는 느낌
- OCR 스캔 입력 (카메라로 찍으면 번호 자동 인식)
- 동행복권 앱 안 켜도 자동 당첨 확인 알림
- 전략별 성과 비교표 (자동번호 대조군과 비교)

## 장기 로드맵

### Phase 1 — 베타 (현재)
- 번호 생성 + 근거 설명 카드
- 구매 이력 + 자동 당첨 확인
- 기본 성과 대시보드

### Phase 2 — 데이터 강화
- 실제 전 회차 데이터 적용
- 주간 자동 업데이트 (GitHub Actions)
- 히트맵 인터랙티브 시각화
- OCR 스캔 입력
- 미출현 기간 차트, 군중 회피 전략 추가
- 번호 입력 격자 UI 교체 (동행복권 용지 스타일)

### Phase 3 — 비주얼 & 배포
- 3D 골든 돼지저금통 히어로 (GLB)
- 모바일 반응형
- 커스텀 도메인
- 주간 AI 브리핑 카드

### Phase 4 — 커뮤니티
- 전략 공유 기능
- 사용자 성과 리더보드 (익명)
- 앱 이름 후보 재검토: LottoMetis / 터닝피그 / GoldenDraw
